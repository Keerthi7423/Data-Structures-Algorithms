## SECTION 1: CONCEPTUAL ANSWERS

1. Difference between var, let, const + hoisting

var:

* Function scoped
* Can be redeclared and updated
* Hoisted and initialized with undefined

let:

* Block scoped
* Can be updated but NOT redeclared
* Hoisted but in Temporal Dead Zone (TDZ)

const:

* Block scoped
* Cannot be updated or redeclared
* Must be initialized at declaration

Hoisting:

* JavaScript moves declarations to the top of scope
* var → initialized as undefined
* let/const → hoisted but not initialized (TDZ)

---

2. Using variable without declaration

* In non-strict mode:
  It becomes a global variable (bad practice)

* In strict mode:
  ReferenceError

3. Data Types in JavaScript

Non-Primitive:

---

4. == vs ===

== (Loose equality):

* Compares value after type conversion

=== (Strict equality):

* Compares value + type (no conversion)

Example:
"5" == 5 → true
"5" === 5 → false

---

5. Type Coercion

* Automatic conversion of one type to another

Examples:
"5" + 2 → "52"
"5" - 2 → 3

---

6. typeof null

Reason:

* Historical bug in JavaScript
* null is not actually an object

7. Dynamic Typing

JavaScript is dynamically typed:

* Variable type can change at runtime

Example:
let x = 10;
x = "hello";
