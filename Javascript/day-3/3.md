DAY-2 JAVASCRIPT NOTES
TOPIC: VARIABLES & DATA TYPES
========================================

----------------------------------------
1) VARIABLE
----------------------------------------
Definition:
A variable is a container used to store data.

Example:
let name = "Keerthi";

Explanation:
- "name" is the variable
- "Keerthi" is the value stored in it

----------------------------------------
2) VAR vs LET vs CONST
----------------------------------------

VAR:
- Function scoped
- Can be redeclared
- Can be reassigned
- Old method (not recommended)

Example:
var x = 10;
var x = 20; // allowed

----------------------------------------

LET:
- Block scoped
- Cannot be redeclared
- Can be reassigned
- Used in modern JavaScript

Example:
let x = 10;
x = 20; // allowed
// let x = 30; ❌ not allowed

----------------------------------------

CONST:
- Block scoped
- Cannot be redeclared
- Cannot be reassigned
- Must be initialized at declaration

Example:
const x = 10;
// x = 20; ❌ Error

----------------------------------------
3) REDECLARATION RULES
----------------------------------------

var → can redeclare ✅
let → cannot redeclare ❌
const → cannot redeclare ❌

----------------------------------------
4) REASSIGNMENT RULES
----------------------------------------

var → can reassign ✅
let → can reassign ✅
const → cannot reassign ❌

----------------------------------------
5) DATA TYPES
----------------------------------------

Definition:
Data types define the type of value a variable holds.

----------------------------------------
6) TYPES OF DATA TYPES
----------------------------------------

PRIMITIVE DATA TYPES:
- string → "Hello"
- number → 123
- boolean → true/false
- undefined → variable declared but not assigned
- null → intentional empty value
- bigint → large numbers
- symbol → unique identifier

----------------------------------------

NON-PRIMITIVE DATA TYPES:
- object → collection of key-value pairs
- array → special type of object

Example:
let arr = [1, 2, 3];

----------------------------------------
7) NULL vs UNDEFINED
----------------------------------------

undefined:
- Variable declared but no value assigned

Example:
let x;
console.log(x); // undefined

----------------------------------------

null:
- Intentional empty value assigned by developer

Example:
let y = null;

----------------------------------------
8) TYPEOF OPERATOR
----------------------------------------

Definition:
typeof is used to check the data type of a value.

Examples:
typeof "Hello" → string
typeof 123 → number
typeof true → boolean

Special Case:
typeof null → object (JavaScript bug)

----------------------------------------
9) IMPORTANT OUTPUT CASES
----------------------------------------

let x = 10;
x = 20;
console.log(x);
Output: 20

----------------------------------------

const x = 10;
x = 20;
Output: TypeError (cannot reassign const)

----------------------------------------

var x = 10;
var x = 20;
console.log(x);
Output: 20

----------------------------------------

let x;
console.log(x);
Output: undefined

----------------------------------------

typeof "Hello" → string
typeof 123 → number
typeof null → object

----------------------------------------
10) WHY LET IS PREFERRED
----------------------------------------

- Block scoped (safer)
- Prevents redeclaration
- Avoids bugs
- Used in modern JavaScript

----------------------------------------
FINAL SUMMARY
----------------------------------------

- Use let for variables
- Use const for fixed values
- Avoid var
- Know all data types
- Understand null vs undefined
- Practice typeof operator
